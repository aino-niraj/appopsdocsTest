# Creating and Managing Projects

Projects in AppOps Builder help organize services, applications, components, themes, and workflows inside a centralized workspace.

Projects act as the primary organizational structure for application development.

---

## Project Structure

A project can contain:

* Services
* Applications
* Pages
* Components
* Themes
* Extensions
* Environments

This structure helps teams manage large-scale applications efficiently.

---

## Creating a Project

To create a project:

1. Open the workspace dashboard
2. Click **Create Project**
3. Enter project details
4. Configure environments
5. Save the project

Example:

```text
Project Name: CRM Platform
Environment: Design
```

---

## Managing Services

Projects can contain multiple services.

Examples:

* HR Service
* CRM Service
* Inventory Service
* Analytics Service

Services help organize related applications and workflows.

---

## Managing Applications

Applications are created under services.

Each application contains:

* Pages
* Components
* Workflows
* Configurations

Applications can be independently managed and deployed.

---

## Environment Configuration

Projects support the AppOps environment lifecycle.

Environment management helps:

* Separate design, testing, and production systems
* Test applications safely
* Manage deployment workflows

Supported environment stages:

* Design
* Test
* Production

---

## Theme Management

Themes can be shared across applications inside a project.

This allows organizations to maintain:

* Consistent branding
* Shared UI systems
* Centralized styling

---

## Team Collaboration

Projects allow teams to work on:

* Shared services
* Shared components
* Shared workflows
* Shared themes

This improves development efficiency and consistency.

---

## Project Organization Best Practices

Recommended practices:

* Group related applications under services
* Reuse components wherever possible
* Maintain centralized themes
* Use environment separation
* Organize pages clearly
* Maintain reusable workflows

Proper project organization improves scalability and maintainability.
